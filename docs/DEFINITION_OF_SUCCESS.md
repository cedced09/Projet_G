# Définition du succès

## Succès du produit

Après quelques semaines d'utilisation, l'application doit permettre de répondre rapidement à ces questions :

- Quelles annonces sont réellement nouvelles ?
- Quels portails publient le même bien ?
- Quels biens correspondent le mieux au projet ?
- Pourquoi un bien est-il bien ou mal classé ?
- Quels renseignements manquent avant de contacter l'agence ?
- Quels critères personnels émergent des décisions précédentes ?
- Quels biens rejetés méritent une nouvelle lecture après un changement d'hypothèse ?

## Succès pédagogique

Le projet doit permettre à l'utilisateur de maîtriser progressivement :

1. création d'une application Python ;
2. conception d'un modèle de données ;
3. migrations et persistance ;
4. interface de consultation ;
5. pipelines d'ingestion ;
6. sorties structurées de LLM ;
7. provenance et validation humaine ;
8. outils appelables par un agent ;
9. orchestration avec état ;
10. déploiement et exploitation.

## Anti-objectifs

Le succès n'est pas :

- maximiser le nombre de frameworks ;
- construire un système multi-agent prématuré ;
- prétendre calculer une rentabilité exacte à partir d'une annonce ;
- automatiser des actions externes sans validation ;
- masquer les hypothèses économiques ;
- collecter toutes les données possibles sans usage défini.
